# 🚀 Установка RU CMS

**Версия:** 2.0  
**Для:** Ubuntu 24.04.3 LTS  
**Уровень сложности:** Простой

---

## 📋 Требования

### Минимальные требования:
- **ОС:** Ubuntu 24.04.3 LTS
- **RAM:** 512 MB (рекомендуется 1 GB+)
- **Диск:** 2 GB свободного места
- **Процессор:** 1 ядро (рекомендуется 2+)

### Программное обеспечение:
- **PHP 8.5** (обязательно)
- **MySQL 8.0+** или MariaDB 10.6+
- **Nginx**
- **Composer 2.0+**
- **Node.js 18+** и npm
- **Git**

---

## 🤖 Автоматическая установка (рекомендуется)

### Шаг 1: Подключение к серверу

Подключитесь к вашему VPS через SSH:

```bash
ssh user@your-server-ip
```

### Шаг 2: Загрузка проекта

```bash
# Перейдите в нужную директорию
cd /var/www

# Клонируйте репозиторий (или загрузите файлы проекта)
git clone <ваш-репозиторий> cms
# или
# Распакуйте архив с файлами проекта в /var/www/cms

cd cms
```

### Шаг 3: Запуск скрипта установки

```bash
# Сделайте скрипт исполняемым
chmod +x install.sh

# Запустите скрипт с правами root
sudo ./install.sh
```

Скрипт установки автоматически:
- ✅ Обновит систему
- ✅ Установит PHP 8.5 и все необходимые расширения
- ✅ Установит MySQL
- ✅ (Опционально) Создаст базу данных и пользователя MySQL автоматически
- ✅ Установит Nginx и настроит конфигурацию
- ✅ Установит Composer
- ✅ Установит Node.js 18+
- ✅ Установит зависимости проекта (Composer и npm)
- ✅ Соберёт frontend ресурсы
- ✅ Настроит права доступа
- ✅ Настроит CRON для задач Laravel
- ✅ Создаст базовый .env файл (с данными БД, если БД была создана автоматически)
- ✅ (Опционально) Установит Redis для кеширования
- ✅ (Опционально) Установит SSL сертификат (Let's Encrypt)

Во время установки скрипт попросит вас:
1. **Создать базу данных MySQL автоматически?** (y/n) - рекомендуется ответить "y"
   - Если вы ответите "y", скрипт попросит ввести:
     - Имя базы данных (по умолчанию: `rucms`)
     - Имя пользователя (по умолчанию: `root`)
     - Пароль пользователя (можно оставить пустым для автогенерации)
   - Скрипт автоматически создаст базу данных и пользователя, а также запишет данные в `.env`
2. **IP адрес или доменное имя** вашего сервера
3. **Установить Redis?** (y/n) - для кеширования
4. **Установить SSL сертификат?** (y/n) - только для доменов (не для IP)

**💡 Совет:** Рекомендуется создать базу данных автоматически во время установки. Это сэкономит время и избежит ручной настройки.

### Шаг 4: Открытие установщика в браузере

**💡 Примечание:** Если вы выбрали автоматическое создание базы данных в Шаге 3, то данные БД уже записаны в `.env` файл, и можно сразу переходить к веб-установщику.

Если база данных не была создана автоматически, вы можете:
- Использовать скрипт `create-database-simple.sh` для создания БД (см. раздел "Решение проблем")
- Или создать базу данных вручную (см. раздел "Решение проблем")
- Или ввести данные БД вручную в веб-установщике

Откройте в браузере:

```
http://your-domain-or-ip/install
```

или (если был установлен SSL):

```
https://your-domain-or-ip/install
```

Следуйте инструкциям установщика CMS, который проведет вас через:
1. **Приветствие** - информация о системе
2. **Проверка требований** - PHP версия, расширения, права доступа
3. **Настройка базы данных** - подключение к БД
   - Если база данных была создана автоматически скриптом, данные уже в `.env` - просто проверьте подключение
   - Если база данных не была создана, введите данные подключения вручную
4. **Создание администратора** - первый пользователь системы
5. **Демонстрация возможностей** - обзор функций CMS
6. **Завершение** - финальная настройка

## ✅ Проверка установки

После завершения установки проверьте:

1. **PHP версия:**
   ```bash
   php -v
   # Должно быть: PHP 8.5.x
   ```

2. **Статус сервисов:**
   ```bash
   sudo systemctl status nginx
   sudo systemctl status mysql
   sudo systemctl status php8.5-fpm
   ```

3. **Доступность сайта:**
   - Откройте `http://your-domain-or-ip` в браузере
   - Должна открыться страница приветствия или установщик

4. **CRON задача:**
   ```bash
   crontab -l
   # Должна быть задача: * * * * * cd /var/www/cms && php artisan schedule:run >> /dev/null 2>&1
   ```

## 🆘 Решение проблем

### Проблема: PHP версия не 8.5

**Решение:**
```bash
# Проверьте установленные версии PHP
ls /etc/php/

# Если PHP 8.5 не установлен:
sudo add-apt-repository ppa:ondrej/php -y
sudo apt update
sudo apt install php8.5 php8.5-fpm php8.5-cli php8.5-mysql php8.5-mbstring php8.5-xml php8.5-curl php8.5-zip php8.5-gd

# Обновите конфигурацию Nginx, чтобы использовать PHP 8.5
sudo nano /etc/nginx/sites-available/rucms
# Измените строку fastcgi_pass на:
# fastcgi_pass unix:/var/run/php/php8.5-fpm.sock;

sudo systemctl restart nginx
sudo systemctl restart php8.5-fpm
```

### Проблема: Ошибки прав доступа

**Решение:**
```bash
cd /var/www/cms
sudo chown -R www-data:www-data .
sudo chmod -R 755 .
sudo chmod -R 775 storage bootstrap/cache
```

### Проблема: Nginx возвращает 502 Bad Gateway

**Решение:**
1. Проверьте, запущен ли PHP-FPM:
   ```bash
   sudo systemctl status php8.5-fpm
   ```

2. Проверьте путь к сокету в конфигурации Nginx:
   ```bash
   sudo nano /etc/nginx/sites-available/rucms
   # Убедитесь, что указан правильный путь:
   # fastcgi_pass unix:/var/run/php/php8.5-fpm.sock;
   ```

3. Перезапустите сервисы:
   ```bash
   sudo systemctl restart php8.5-fpm
   sudo systemctl restart nginx
   ```

### Проблема: Ошибки при установке зависимостей

**Решение:**
1. Убедитесь, что у вас установлен Composer:
   ```bash
   composer --version
   ```

2. Если Composer не установлен:
   ```bash
   curl -sS https://getcomposer.org/installer | php
   sudo mv composer.phar /usr/local/bin/composer
   ```

3. Очистите кэш Composer:
   ```bash
   composer clear-cache
   cd /var/www/cms
   composer install --no-dev --optimize-autoloader
   ```

### Проблема: Не могу создать базу данных MySQL

**Решение:**

Если база данных не была создана автоматически во время установки, используйте скрипт `create-database-simple.sh`:

```bash
cd /var/www/cms
sudo chmod +x create-database-simple.sh
sudo ./create-database-simple.sh
```

Этот скрипт автоматически:
- Проверит и установит MySQL, если его нет
- Создаст базу данных и пользователя
- Сохранит данные в файл `mysql_credentials.txt`

**Альтернативный способ - создание вручную:**

Если `sudo mysql` не работает (требует пароль), используйте скрипт выше или сбросьте пароль:

```bash
# Остановите MySQL
sudo systemctl stop mysql

# Запустите MySQL в безопасном режиме
sudo mysqld_safe --skip-grant-tables --skip-networking &
sleep 3

# Подключитесь без пароля
mysql -u root <<EOF
USE mysql;
FLUSH PRIVILEGES;
ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY 'новый_пароль';
FLUSH PRIVILEGES;
EOF

# Остановите безопасный режим
pkill mysqld_safe
sleep 2

# Запустите MySQL
sudo systemctl start mysql
```

### Проблема: Ошибка доступа к базе данных в веб-установщике

**Решение:**
1. Проверьте, что база данных создана:
   ```bash
   sudo mysql -e "SHOW DATABASES;"
   ```

2. Проверьте права пользователя:
   ```bash
   sudo mysql -e "SHOW GRANTS FOR 'rucms_user'@'localhost';"
   ```

3. Проверьте данные в `.env`:
   ```bash
   cd /var/www/cms
   cat .env | grep DB_
   ```

4. Убедитесь, что используется правильный драйвер сессий для установки:
   - В `.env` должно быть: `SESSION_DRIVER=file`
   - Это настраивается автоматически скриптом установки

## 🔒 SSL сертификат (Let's Encrypt)

Если вы установили SSL через скрипт, сертификат будет автоматически обновляться через системный таймер Certbot.

**Установка SSL после основной установки:**
```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com -d www.your-domain.com
```

**Проверка автообновления SSL:**
```bash
sudo systemctl status certbot.timer
```
